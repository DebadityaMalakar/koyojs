CHANGELOG
=========

[0.2.2] - 2026-06-30
---------------------
Object utilities expanded — honest cloning split, nested-path pick/omit.

New functions (src/Object):
  - cloneShallow    Wraps structuredClone. Fast, but throws on non-serializable
                    values (functions, symbols) and drops prototypes on class
                    instances. Renamed from the old cloneDeep.
  - cloneDeep       Rewritten as a proper recursive walk. Preserves prototypes
                    (instanceof stays intact), clones Map and Set, handles
                    circular references via WeakMap, and keeps getters/setters
                    as descriptors instead of flattening them to values.
  - extract         Like pick, but resolves dot-notation nested paths.
                    extract(obj, ['a.b.c']) returns { a: { b: { c: value } } }.
  - exclude         Like omit, but resolves dot-notation nested paths.
                    exclude(obj, ['a.b.c']) removes the leaf, leaving siblings
                    and the parent key intact.

Benchmark fixes:
  - pick/omit now only compare against libraries doing the same operation
    (top-level keys). lodash is moved to the extract/exclude groups where the
    path-capable comparison is actually apples-to-apples.
  - cloneDeep and cloneShallow are in separate benchmark groups so the chart
    stops conflating the two different feature sets.

[0.2.1] - 2026-06-30
---------------------
Performance — speed regressions fixed, benchmark suite added.

Performance fixes:
  - toTitleCase     Rewrote from String.replace(regex, callback) to a single-
                    pass charCodeAt loop tracking word-boundary state. Eliminates
                    a closure allocation per word match. ~2x faster on short
                    strings, measurably faster on long strings.
  - chunk           Rewrote from slice()+push() to preallocated outer and inner
                    arrays with a manual element-copy loop. Both array sizes
                    computed upfront via Math.ceil; no repeated reallocation.
                    ~28% faster at all tested sizes.

Benchmark suite (benchmark/):
  - benchmark:node     node benchmark/node.mjs   — imports from dist/
  - benchmark:bun      bun  benchmark/bun.ts     — imports from src/ (native TS)
  - benchmark:browser  node benchmark/browser.mjs — dist/ + happy-dom globals
  - benchmark:all      runs browser → node → bun in sequence
  Compares koyojs against lodash, remeda, and radash across all functions.

[0.2.0] - 2026-06-30
---------------------
Array and Object utilities.

Array utilities (src/Array):
  - createArray     Typed array wrapper. strict:true throws TypeError on type
                    mismatch; strict:false emits a console.warn. Type is inferred
                    from the first pushed element, or from a custom validate fn.
  - unique          Remove duplicate items from an array.
  - removeDuplicates  Alias for unique.
  - flatten         Recursively flatten a nested array to a single level.
  - chunk           Split an array into sub-arrays of a given size.
  - groupBy         Group items by a key derived from a function.

Object utilities (src/Object):
  - pick            Return a new object with only the selected keys.
  - omit            Return a new object with the specified keys removed.
  - cloneDeep       Fully independent deep copy via structuredClone.

[0.1.1] - 2026-06-29
---------------------
Bug fixes.

- fix: subpath imports now work — "koyojs/String" and "koyojs/Number" are
  properly declared in package.json exports with separate ESM (.mjs) and CJS
  (.cjs) bundles emitted for each. Previously only the root "." entry existed,
  causing ERR_PACKAGE_PATH_NOT_EXPORTED at runtime.
- fix: ellipsify README description corrected — maxLength is the cap on total
  output length including the "...", not the cap before appending it.
  ellipsify("Hello World", 5) → "He...", not "Hello...".

[0.1.0] - 2026-06-29
---------------------
Cross-environment compatibility pass.

- TypeScript build config now uses lib: ES2020 + DOM, no ambient Bun types in
  emitted declarations. Published types are self-contained and work in any TS
  project regardless of runtime.
- Test suite now runs in both Node and a browser-like environment (happy-dom)
  to verify behaviour is identical across runtimes.
- Version bumped from 0.0.1 to 0.1.0.

[0.0.1] - 2026-06-29
---------------------
Initial release.

String utilities (src/String):
  - capitalize      Capitalize the first letter of a string.
  - toCamelCase     Convert a string to camelCase.
  - toSnakeCase     Convert a string to snake_case.
  - toTitleCase     Convert a string to Title Case.
  - truncate        Shorten a string to a maximum length.
  - ellipsify       Shorten a string and append "..." when truncated.
  - slugify         Convert a string to a URL-friendly slug.
  - isEmail         Validate an email address format.
  - isURL           Validate an HTTP/HTTPS URL.
  - isPhoneNumber   Validate a phone number (E.164 and common formats).

Number utilities (src/Number):
  - clamp           Restrict a number to a [min, max] range.
  - randomInt       Generate a random integer within a [min, max] range.
  - sum             Sum an array of numbers.
  - average         Average an array of numbers.
  - isEven          Check whether an integer is even.
  - isOdd           Check whether an integer is odd.
